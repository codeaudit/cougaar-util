Default COUGAAR Components

The following components are hard-coded to load in node-agents and standard agents.

See bug 2522 for Cougaar 10.2 cleanup ideas. This list was created for Cougaar 10.0 (2/1/2003).

All components build upon this simple loading template: 

  initialize:

  load(PRIORITY_HIGH):
            < ".ini" PRIORITY_HIGH >

  load(PRIORITY_INTERNAL):
            < ".ini" PRIORITY_INTERNAL >

  load(PRIORITY_BINDER):
            < ".ini" PRIORITY_BINDER >

  load(PRIORITY_COMPONENT):
            < ".ini" PRIORITY_COMPONENT >

  load(PRIORITY_LOW):
            < ".ini" PRIORITY_LOW >

  start:

SimpleAgent, the base class of all "cluster=" agents, extends the above component skeleton.

Here is the hard-coded loading sequence for SimpleAgent: 

  initialize:
        < setupMessageHandlers >

  load(PRIORITY_HIGH):
        AgentIdentificationService
        EventService
            < ".ini" PRIORITY_HIGH >

  load(PRIORITY_INTERNAL):
        < acquire agent identity >
        < register in topology >
        < register in mts >
        AlarmService
        DemoControlService
        MessageSwitchService
            < ".ini" PRIORITY_INTERNAL >

  load(PRIORITY_BINDER):
            < ".ini" PRIORITY_BINDER >

  load(PRIORITY_COMPONENT):
        AgentContainmentService
        UIDService
        ThreadService  (as non-root)
        SchedulerService
        PrototypeRegistryService
        LDMService
        ServletService (override for agent)
        DomainService              (from domain)
        DomainForBlackboardService (from domain)
        < domain loads "planning" factory >
        CommunityService
        BlackboardForAgent       (from blackboard)
        BlackboardService        (from blackboard)
        BlackboardMetricsService (from blackboard)
        BlackboardQueryService   (from blackboard)
            < ".ini" PRIORITY_COMPONENT >
        < obtain BlackboardForAgent, for message handling >

  load(PRIORITY_LOW):
            < ".ini" PRIORITY_LOW >
        PluginManager  (loads all "plugin=" components)

  start:
        < start message queue >
        < reconcile with other agents >
        < launch agent restart checker >

The NodeAgent loading sequence is similar, since NodeAgent extends SimpleAgent.

All single-indent lines are specific to NodeAgents, and the double-indent lines are common to the above SimpleAgent loading sequence. 

  <Node constructor>:
    NodeIdentificationService
    ComponentInitializerService (as specified by system properties)
         (Optional: DBInitializerService, as needed by the ComponentInitializerService)
    AgentManager

  initialize:
        < setupMessageHandlers >

  load(PRIORITY_HIGH):
    NodeControlService
    LoggingService
        AgentIdentificationService
        EventService
            < ".ini" PRIORITY_HIGH >
    AgentIdentityService  (default impl)

  load(PRIORITY_INTERNAL):
    ThreadService  (as root)
    NamingService
    WhitePagesService
    TopologyWriterService
    TopologyReaderService
    MetricsUpdateService  (from qos)
    MetricsService        (from qos)
    NodeMetricsService
    MessageTransportService  (from mts)
    MessageStatisticsService (from mts)
    MessageWatcherService    (from mts)
    AgentStatusService       (from mts)
    < mts-internal aspects >
    NaturalTimeService
    RealTimeService
    ServletService  (as root)
    NodeTrustService
        < acquire agent identity >
        < register in topology >
        < register in mts >
        AlarmService
        DemoControlService
        MessageSwitchService
            < ".ini" PRIORITY_INTERNAL >

  load(PRIORITY_BINDER):
            < ".ini" PRIORITY_BINDER >

  load(PRIORITY_COMPONENT):
    CommunityInitializerService (unless already loaded, or disabled by -D argument)
    AssetInitializerService (unless already loaded, or disabled by -D argument)
        AgentContainmentService
        UIDService
        ThreadService  (as non-root)
        SchedulerService
        PrototypeRegistryService
        LDMService
        ServletService (override for agent)
        DomainService              (from domain)
        DomainForBlackboardService (from domain)
        < domain loads "planning" factory >
        CommunityService
        BlackboardForAgent       (from blackboard)
        BlackboardService        (from blackboard)
        BlackboardMetricsService (from blackboard)
        BlackboardQueryService   (from blackboard)
            < ".ini" PRIORITY_COMPONENT >
        < obtain BlackboardForAgent, for message handling >

  load(PRIORITY_LOW):
            < ".ini" PRIORITY_LOW >
        PluginManager  (loads all "plugin=" components)

  load(end):
    < emptyQueuedMessages >
    < add agents >

  start:
        < start message queue >
        < reconcile with other agents >
        < launch agent restart checker >
