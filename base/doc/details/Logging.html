Generic Logging Facility

COUGAAR 8.6
  • Two Logging Services Implemented: Logging and LoggingControl
    • Logging service used by developers to mark software events (org.cougaar.core.logging.LoggingService)
    • Logging Control service used to control logging output (org.cougaar.core.logging.LoggingControlService)
    •  
  • Log4j serves as the backend for both services
    •  
  • Logging service has two axis: Object doing logging and Severity level
    • First you request a LoggingService from the ServiceBroker
    • Request marks the object doing the logging
    • Calls to the logging service take the form objectLog.debug("String")
    • Five levels of severity: debug, info, warn, error, fatal
    •  
  • Standardized logging system allows Cougaar developers to log software events without developing their own logging tools
  • Standardized logging controls via PSP allows Cougaar users to control the logging dynamically at runtime even in parts of the software the are unfamiliar with
  • A request to the node level ServiceBroker will create a logging service for the component.
