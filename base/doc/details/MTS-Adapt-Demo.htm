Message Transport Adaptive Behavior Demo

 

 

  1. Introduction

The MTS adaptive code provides an adaptive message transport behavior to the society.  The adaptivity refers to the ability to use compression, SSL message encryption and traffic masking under different threat conditions and network bandwidth. 

 

For purposes of demonstration we have created four levels of threat: Compromised, Suspect, Normal and Carefree.  This threat level can be set as a society parameter at startup or the threat level can be changed in a running society via the PanicButtonPlugin.  This plugin currently sends a threat level policy to all Nodes running in the society.  This trust level information is then processed by a node level component, which provides the trust level to the message transport layer via a service.  In the future, the trust levels may vary for subnets, specific hosts, etc instead of one level applying to an entire society.  We also hope to have society monitors that will dynamically change the trust level as threats are detected.

 

RSS (Resource Status Service) provides a unified way of accessing the status of system resources.  Instances of RSS are local to each Node.  The Node queries RSS and gathers system information such as load average, JIPS (Java Instructions Per Second) and bandwidth between machines.

 

There are two ways in which RSS delivers the information:

Statically:  It reads the values from the three files: Sites.conf, Hosts.conf and TestIP.conf, from a web server or from local files.

Dynamically:  Using STATUSTEC or REMOS  (not integrated at this point).

  1. Requirements

2.1   Full Cougaar Install

1.       JDK1.3      

2.       mySQL 3.23.45 version

3.       cougaar.zip and cougaar-support.zip

4.       cougaar.rc in $COUAAR_INSTALL_PATH/configs/common or .cougaarrc in $HOME

2.2   Configuration Files for Quo and RSS

·         An SSL keystore should be in $COUGAAR_INSTALL_PATH on each machine that will participate in the Cougaar society.

·         tcpdump version 3.6.  At the TIC, u117e has tcpdump version 3.  eiger.alpine.bbn.com/~psharma/tcpdump. 
(Running tcpdump requires root permission on the machine).

·         Modify the appserver property file ($COUGAAR_INSTALL_PATH/server/bin/server.props) to include the SSL flag, SSL keystore, and the property that specifies the adaptdemo.props file.  Use $COUGAAR_ISNTALL_PATH/quo/data/server.props as an example.

·         $COUGAAR_INSTALL_PATH/quo/data/adaptdemo.props specifies the kernel.conf file, a switch for the QUO Kernel GUI describes, various link protocols, aspects,
debugging flags, compression switches and traffic masking capabilities

·         $COUGAAR_INSTALL_PATH/quo/data/kernel.conf is a configuration file for RSS.  It provides both compile time and run time capabilities.  At this time, only the compile time capabilities are being used.  The compile time files are read from a url from a web server or a local file.  There are three of these configuration files: Hosts.conf, TestIP.conf and Sites.conf.

·         $COUGAAR_INSTALL_PATH/quo/data/Hosts.conf describes the static loadavg, Jips and cpu count for each machine participating in the society.

·         $COUGAAR_INSTALL_PATH/quo/data/TestIP.conf describes the max and unused bandwidth between a pair of machines.  This file contains specific machine to machine values, which can override generic values for subnets found in Sites.conf.

·         $COUGAAR_INSTALL_PATH/quo/data/Sites.conf describes the bandwidth between the two subnets for all the machines in the subnet(s) where the society will run.  Note that for demonstration purposes, bandwidth between enclave 3 and enclave 4 at the TIC have been set at a low bandwidth in order to force compression of traffic between these two enclaves.

·         $COUGAAR_INSTALL_PATH/quo/data/README.txt describes all of the configuration files and parameters found in the quo/data directory.

 

 

 

  1. Running the Engineering Assessment

 

3.1  Inserting the PanicButtonPlugin

To run with MTS Adaptive Behavior configure a Cougaar society with the requirements listed in section 2.  Make sure that you have SSL turned on and a valid keystore that is referenced in your server.props file.  In order to actively change the trust level during a society run, you must add a recipe for the PanicButtonPlugin while configuring your society in CSMART.  The following paragraph provides detailed steps for creating and adding the plugin recipe.

Start CSMART.

Right click on Default Workspace

Select NewRecipe and Built In

For Recipe Type, select Specific Insertion and click the OK button

Enter a new name for the insertion such as PanicButtonPlugin_Insertion and click the OK button.

Right click on your newly created Insertion and then click on Configure

A Configuration Builder window will appear with your insertion listed on the left.

Click on your insertion and a table of arguments will appear on the right.

For the Class Name value type in org.cougaar.lib.quo.PanicButtonPlugin

For the Component Name value type in org.cougaar.lib.quo.PanicButtonPlugin

Leave the Number of Arguments value at 0

For Target Component Selection Query, select recipeQueryNCAAgent from the pulldown menu in the Value field.

For the Type of Insertion value type in plugin (make sure to type in a return)

Then select the File Menu and the Save To Database option.

Once the recipe has been successfully saved to the database select File, Close.

When you configure your experiment using CSMART – drag your PanicButtonPlugin_Insertion into the experiment’s configuration window under recipes. 

 

Your experiment will now run with this plugin in the NCA Agent – a PanicButtonPlugin Gui will appear when you start your society.  To manually change the trust level of a society, choose a trust level and then click on the Create and Send New NodeTrustPolicy button.

 

 

 

 

 

3.2  Looking at the QUO Kernel GUI

The Quo Kernel Gui is turned on and off by a parameter in the adaptdemo.props file:

org.cougaar.lib.quo.kernel.gui=true.  One Gui per Node will appear during society startup.  To reduce the number of Gui’s, you may set the Quo Kernel Gui property to false for some nodes and to true for others if you are running off of a local COUGAAR_INSTALL_PATH for each participating machine.

 

The Quo Kernel Gui will refresh frequently during the startup of the society as it collects information about each link to each remote Node.  Choose a top level Kernel Gui for one of the Nodes.

 

The interesting information for these tests can be found in the Gui in the three contracts under Contracts tab.  Adjust the Gui window to the right to view the contracts.

org.cougaar.lib.quo.TrafficMask: one contract per Node

org.cougaar.lib.quo.RemoteSSL: one contract per Agent

org.cougaar.lib.quo.Compress: one contract per Agent

 

Focus on the TrafficMask contract by expanding its tree.

 

 

Choose one of the TrafficMasking Nodes to view the contract between the Node listed on the title bar of the Kernel Gui and the Node listed on the contract.  For example, in this set of screen shots we are looking at the SMALL_1AD Node and the SMALL_1AD_PART Node.  Double click on the TrafficMask node you wish to look at (in our example Traffic_SMALL_1AD_PART(Node)) and another window will appear.

 

 

 

The top left panel shows the bandwidth between the hosts and the bottom left panel shows the observed Trust Level value.  By default each society starts at Trust Status 10, which is ‘Carefree’.  Note that the left middle panel shows UseMasking = false, this panel reflects whether TrafficMasking has been forced – for our purposes, this panel can be ignored.

 

At the right you will see a panel with the AdaptiveTrafficMask title and the Carefree section highlighted.  Also note that SameHost, SameLan or Wan will be highlighted depending on the relationship between the two nodes. If you are running with RSS, this information will come from the RSS configuration files; otherwise it will use a default SameLan Value.

 

The TrustObserver panel and the AdaptiveTrafficMask panel will dynamically change as the message transport components respond to changing threat levels. As previously mentioned, the PanicButtonPlugin Gui should be used to alter the threat conditions.

 

 

3.3  Changing the Trust Level and Viewing the Results

Once you have your society up and running with all of the Quo Kernel Guis and the PanicButtonPlugin, start the society as usual by sending the Oplan and then the GLS task.  Your Quo Gui should still show the Trust Level as 10 and running in the Carefree mode.  The easiest way to check out the messages being passed between Nodes is to use tcpdump.  The following tcpdump command worked well for us, but feel free to optimize the parameters as needed.

 

tcpdump -Xa -s 300 tcp and not port 6011 and not port ssh and not port mysql and  host <u000e> |grep cougaar

where, the output format is:
-Xa  hex and asscii dump
-s 300   first 300 bytes of tcp packet
(just a little of the data field)

and the filter is:
tcp  (only tcp protocol)
not port 6011 (ssh xwindow forwards, sometimes xwindows 6000)
not port ssh  (not ssh shell)

not port mysql  eliminates call to the database mysql
host <a> and host <b>  (source and destination hosts)

| grep cougaar is optional but it can reduce the amount of data to look at

 

In Carefree mode you should see lots of traffic that is pretty human readable.  In fact if you add the | grep cougaar to the end of your tcpdump you will see lots of messages with the org.cougaar.core, org.cougaar.domain, etc classpaths.

 

Now change the Trust Level via the PanicButtonPlugin by choosing a new Trust Level and clicking on the Create and Send New NodeTrustPolicy button.  You should see the Quo Gui panels change almost immediately.  For example, if I change the Trust Level to 0 – Compromised and send the new policy in the PanicButtonPlugin, the Quo Gui should look like the screenshot below:

 

 

 

 

Reference the table in section 1 to determine which adaptive conditions should be turned on depending on the network between the 2 nodes you are watching and the current value of the Trust Level. 

 

You should expect to see the following when looking at tcpdump output:

Carefree level:  Lots of readable output with and without the |grep cougaar argument.

With Compression On: Lots of compressed output viewable by adding a | grep \\[B to your tcpdump arguments

With TrafficMasking On:  You should see lots of extra messages being sent.  If your society has already quiesced, you should still see lots of message traffic.

With SSL On: Lots of unreadable (encrypted) output.

We have noticed that if you use | grep cougaar with SSL on, there are still a few readable messages that always start with org.cougaar.tools when running with CSMART.  We believe this is CSMART communicating with the AppServers, as these messages do not appear when running societies without CSMART.

 

  1. Running MTS Adaptive Behavior for the Assessment

For the Assessment, the MTS Adaptive Behavior should be run at the Compromised level for the entire Society run.  The Trust Level should NOT be changed for these experiments; therefore the PanicButtonPlugin is not needed.

 

Assessment preparation checklist:

- Make sure the society has everything contained in Section 2 Requirements.

- Edit the $COUGAAR_INSTALL_PATH/server/bin/server.props to uncomment the following property:

org.cougaar.ndoe.trustStatus=0

This property will cause the society to run in Compromised mode.

- Turn of the Quo Kernel Gui by editing the Gui property in the $COUGAAR_INSTALL_PATH/quo/data/adaptdemo.props file to read:

org.cougaar.lib.quo.kernel.qui=false

- Do not add the PanicButtonPlugin recipe to the experiment

 

  1. Running MTS Adaptive Behavior without RSS

If you need to run without the RSS Module comment out the property that specifies the location of the kernel.conf file in the adaptdemo.props file:

#org.cougaar.rss.properties=/opt/cougaar/quo/data/kernel.conf

This will cause the MTS adaptive behavior to assume that all Nodes of the society are on the same LAN and the bandwidth will default to 10000.  You will see a message in the Node window stating the No rss URL was found.
