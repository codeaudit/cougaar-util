Archiving Test

The archiving test demonstrates that a Cougaar society can be run while deleting old tasks and later views can be obtained based on data that existed in the past, but has been deleted from the running society. The procedure follows. We are shooting at a moving target in this test since replanning can happen at any moment. A significant portion of this procedure attempts to get the target to hold still while its picture is being taken with two different cameras: an inventory chart and an archive snapshot. There is no guarantee of success, but the chances are good.

Test Procedure

1.        Clear old persistence data (if any)

2.        Start the augmented small-135 society. The society has been augmented with a DeletionPlugIn in every agent. The DeletionPlugIn parameters call for a deletion run every day at midnight GMT. The default deletion delay is 2 days, but that is typically overridden by the inventory manager. Also archiving is enable in the 3-FSB agent. Archiving is disabled in the other agents to save time and disk space.

3.        Set the OPLAN and send GLS.

4.        Start the event generator. The event generator uses its default plugins so consumption will, on average, equal the expected rate.

5.        Advance time with the event generator to 8/15/05 23:45 in 200 seconds. 8/15 is C-day. This is a fast rate, but very little happens before C-day.

6.        Advance 10 minutes to 23:55 and pause. This allows the accumulated consumption reports to be sent from the event generator and processing to commence.

7.        Disable Auto Send of F/C Reports and Inventory Reports in the event generator. The suppresses additional reports from being delivered to the society and minimizes the probability of things changing

8.        Wait for quiescence.

9.        Fire up the InventoryChartUI and examine the inventory of JP8 at the 3-FSB. Zoom in to the initial period up to about September 30 and turn off “Show Inactive”.

10.     Take a screen snapshot.

11.     Advance time 10 minutes and wait for quiescence. Archiving and deletion should occur plus response to the deletion.

12.     Observe the contents of the persistence directory ($COUGAAR_INSTALL_PATH/P/3-FSB). The latest sequence_nnnnn file corresponds to the archive just completed. Because we are running at an accelerated rate, the creation time of the file cannot be used to pick the correct archive.

13.     Advance time 4 days to 8/19/05 23:50 as above. This time run at about 3 minutes per day or 12 minutes. This allows the society to keep up with the planning needs for the interval.

14.     Advance time to 23:59 and pause again.

15.     Update the inventory chart. Observe the loss of some demand and refill tasks. (Projected demand remains because the end time of the projection task has not been reached.)

16.     Take screen snapshot.

17.     Terminate the society.

18.     Start the VIEW society using VNode. The VIEW society has the agents of the original society, but all agents are devoid of plugins except for the PlanServerPlugIn. The VNode script takes a single argument: the suffix of the sequence_nnnnn file recorded above.

19.     Fire up the InventoryChartUI again and examine the inventory of JP8 at the 3-FSB. Zoom in as before.

20.     The chart should be the same unless consumption and replanning occurred in that last minute of the day before the archive was created.
