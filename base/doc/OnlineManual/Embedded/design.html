Design and Architecture

The design of embedded Cougaar either removes or replaces (with a trivial implementation) the standard Node-level services. Services depend on jar files and sometimes config files. So the Embedded Couggar installation will need all the files that will be used by the society configuration.

The following subsystems can be reduced or eliminated:

  • HTTP Logistics/Planning
  • HTTP Servlets
  • Communities
  • Yellow Pages
  • Message Transport Service
  • Metrics Service
  • Thread Service
  • Blackboard
  • Persistence

This writeup will use three example Embedded configurtions:

  • Minimum Configuration: Replaces or removes all the standard services.
  • Standard Configuration: All standard services, without metrics, planning, and communities.
  • Metrics Configuration: Standard configuration with addition of Metrics service.
