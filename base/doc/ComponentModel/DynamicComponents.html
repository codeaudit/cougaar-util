Dynamic Components:

The initial API for the dynamic creation and mobilitity of Components has
been created.  In particular, a new "ComponentMessage" was created which
allows one to specify:
  - an operation: Add / Remove / Suspend / Resume / Reload
  - the MessageAddress where this command is to be sent
  - the ComponentDescription of the Component to Add/Remove/etc
The ComponentDescription can specify an optional "parameter", which will
allow mobile components to specify their "state" prior to movement.  Other
higher-level Messages may be created in a future release, such as a
"MoveComponentMessage" and atomic-transaction/security support.

Currently only Nodes and Clusters are MessageTargets, so one can only
send these message to Nodes (to add/remove/etc a Cluster) and to Clusters
(to add/remove/etc a Plugin).  Plugins will be provided a MessageTransport
service which will allow Plugins to create and send these messages.

For the initial implementation only the addition of new Clusters and Plugins
is supported until the requisite Container support is provided in the
infrastructure (namely Node and ClusterImpl).  Developers can create and
send any ComponentMessage, but for now non-"add" ComponentMessages will
generate a warning and be ignored.
 

Console/Server redesign:

The remote creation and management of Nodes, i.e. the "NodeServer" and it's
GUI "Console", have been redesigned:
  - Separated RMI code from client (Console)
  - Created a simple three-interface API for console developers,
      with all RMI-specific code in the "rmi" package
      (details in the package-level javadocs:
       "cougaar/server/src/org/cougaar/tools/server/index.html")
  - Renamed RMI-specific code to follow easier-to-understand
      naming conventions (details in the package-level javadocs:
      "cougaar/server/src/org/cougaar/tools/server/rmi/index.html").
  - Added Node registration back to the server, which allows the
      client (Console) to directly contact the Node
      (e.g. ask the Node for a List of all it's ClusterIdentifiers)
  - Added a "listener" pattern for the client (Console) to receive
      notification of events within the Node
  - Refactored Console to be a simple example of a server-client --
      in the future it should be removed from this package.
These modifications will allow for a more powerful Console to be created with
greater control over the Nodes.  These modifications also required the
addition
of three core "ExternalNode*" classes to
"cougaar/core/src/org/cougaar/core/society" and the modification of Node to
create and register an RMI "hook" for the remote server.  For the typical
command-line Node invocation these modifications are disabled.
 

Dynamic PSP load/unload:

The PSP "?LOAD" and "?UNLOAD" capability has been enhanced.  These
PSP-services
now produce better HTTP interfaces for browsers, and the URL parameters have
been renamed for easier use:
  ?LOAD?PSP_PATH=Path?CLASSNAME=JName?LOADER=URL
     - Path: logical PSP location at this server (must end with .PSP)
     - JName: Java class name of 'Plan Service Provider'
     - URL: Class is loaded from this URL (optional -- default uses
server’s classpath)
  ?UNLOAD?PSP_PATH=Path?CONFIRM_UNLOAD=Confirm
     - Path: logical PSP location at this server (must end with .PSP)
     - Confirm: Use "Yes", otherwise the user is prompted for a yes/no value
The "?LOADER" parameter of the PSP "?LOAD" now correctly loads PSP from remote
URL locations.
