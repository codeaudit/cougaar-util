Below are directions for Running the SMALL-135 Society across one or two hosts:

Get the build

Download the latest build of the Cougaar software off the Alpine website(www.alpine.bbn.com) on two machines in your network.
2 .zip files you need to run the Small-135 society:
        # cougar.zip
        # cougar-support.zip
 

Unzip and install

In a 2-host society, the Demand Community and Transportation Community will be split across two machines.

Unzip and install the society on each machine:

    #  Make sure that COUGAAR_INSTALL_PATH is set properly to the top release directory
    #  unzip cougar.zip
    #  unzip cougar-support.zip   (answer whatever to its question)
    #  edit $COUGAAR_INSTALL_PATH/configs/common/alpreg.ini to make Demand host common - (set to localhost for one host society, else choose one machine's full address)
    # check the MYPROPERTIES environment variable in $COUGAAR_INSTALL_PATH/bin/Node (for linux) or setarguments.bat(for windows) to make sure it contains "-Dorg.cougaar.core.cluster.startTime=08/10/2005"

Run Society

Start the Demand Community on the first machine:

  1. Make sure that COUGAAR_INSTALL_PATH is set properly to the top release directory
  2. cd to $COUGAAR_INSTALL_PATH/configs/small-135/
  3. Type Node SMALL-135-NODE
Start the Transportation Community on the second machine/same machine in one host society:
  1. Make sure that COUGAAR_INSTALL_PATH is set properly to the top release directory
  2. cd to $COUGAAR_INSTALL_PATH/glmtrans/configs/demo-config/configs
  3. Type Node GLMTransNode
Since the Transportation Community has been greatly simplified, and is now much smaller than in previous releases (7 clusters versus 37 clusters), it should be possible to run both nodes in a single machine, provided there is enough memory (at least about 400 MBytes).

Collecting Results

Inventory graphs and TPFDD's can be produced to provide analysis of the society's run.

To run an Inventory graph:

    #  cd to $COUGAAR_INSTALL_PATH/bin
    #  Type CUI_Inventory
    #  Set the Them to "COUGAAR"
    #  Choose the connection "localhost:5555"
    #  Choose an Organization
    #  Choose a Commodity

(press 'Back' on this brower's menu to see some example Organizations and Commodity expected results)

To run Strategic and InTheater TPFFDs:

    - you must edit two xml files in order to run the Strategic and InTheater data grabbers (on each host that will run the TPFFD):

        #  cd to $COUGAAR_INSTALL_PATH/datagrabber/bin
        #  mv InTheaterGrabberConfig.xml InTheaterGrabberConfig.xml.orig
        #  cp InTheaterGrabberConfigExample.xml InTheaterGrabberConfig.xml
        #  mv StrategicGrabberConfig.xml StrategicGrabberConfig.xml.orig
        #  cp StrategicGrabberConfigExample.xml StrategicGrabberConfig.xml

    - now you may run the grabbers:

        In UNIX:
        #  tcsh strategic_datagrabber.csh
        #  tcsh intheater_datagrabber.csh

    - you must edit the file 'runNewTPFDDShell.csh' to configure your system properly, so that you may access the data gathered in an http browser:

        #  Type emacs& runNew TPFFDShell.csh
        #  Change these 4 varaibles - 'somehost' is your specific Demand host:

                -Dorg.cougaar.domain.mlm.ui.newtpfdd.producer.ClusterCache.deaultHostName=somehost
                                                                             "...          .DataGrabberProducer.database=grabber
                                                                                           "...                               .dbUser=root
                                                                                           "...                               .dbPassword=vishnu
        #  save the file

    - run the new tpfdd shell:

        #  tcsh runNewTPFDDShell.csh

    - in your http browser [while new tpfdd shell is running]:

        #  go to http://localhost:7000       - this will list all the options for viewing output available and many other statistics and queries
